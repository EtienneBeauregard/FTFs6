Mircro service de test avec API REST simple.

Ce microservice de test peut être utilisé pour tester les fonctionalités de base d'un API REST. Quatre méthodes Http ont été implantés: GET, POST, PUT et DELETE. L'API se compose d'une collection de ressources Student accessible via l'URI (inclure votre chemin local avant) l'URI de la ressource):

  • students

Chaque étudiant possède un nom et un CIP. Chaque ressource représentant un étudiant est disponible via l'URI

  • students/{cip}

Voici maintenant une liste de requêtes que vois pouvez effectuer pour tester le système. Vous pouvez utiliser un outil comme Postman (Chrome) ou RESTClient (Firefox) pour ce faire.

Inspecter les étudiants avec GET

Pour avoir une liste complète de tous les étudiants disponibles dans la base de données:

  • students

Pour un étudiant seul, disons François avec le CIP pouf1903:

  • students/pouf1903

Ajouter des étudiants avec POST

Ajoutons un étudiant nommé Marcel avec le CIP gmar2104. D'abord, dans l'en-tête il faut spéficier le type de contenu: Content-Type : application/json. Dans le corps de la requête, on ajoute les informations de Marcel en JSON: { "cip": "gmar2104", "name": "Marcel" }. On envoie ça à l'URI:

  • students

Vous pouvez visualiser le résultat avec un GET.

Modifier le nom des étudiants avec PUT

Modifions le nom de l'étudiant Éric, avec le CIP poie2104, pour Rolland. D'abord, dans l'en-tête il faut spéficier le type de contenu: Content-Type : application/json. Dans le corps de la requête, on ajoute les informations à jour (même CIP, nouveau nom) au format JSON: { "cip": "poie2104", "name": "Rolland" }. On envoie ça à l'URI:

  • students

Vous pouvez visualiser le résultat avec un GET.

Supprimer des étudiants avec DELETE

Modifions le nom de l'étudiant Éric, avec le CIP poie2104. Pour ce faire, il suffit d'appeler DELETE sur l'URI:

  • students/poie2104

Vous pouvez visualiser le résultat avec un GET.
